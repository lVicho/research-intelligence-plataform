# Documentation Instructions

## Scope

Applies to `docs/**` and documentation-only changes elsewhere. Also follow root `AGENTS.md`.

## Rules

- Keep architecture and implementation docs factual and tied to the current repository state.
- Do not invent completed features, metrics, integrations, performance claims, or model quality claims.
- Spanish demo/product docs should be in Spanish.
- Architecture and Codex task-context docs may be in English unless a task requests Spanish.
- Prefer concise summaries with links or file references over long duplicated guidance.
- Document limitations and pending work explicitly.
- Do not add secrets, real credentials, private data, or generated large files.
